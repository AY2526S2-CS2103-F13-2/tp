Project Overview

TripLog is a CLI-centric desktop application designed for travelers who prefer managing their itineraries via a terminal interface. It allows users to track destinations and trip schedules, providing a streamlined, keyboard-first alternative to traditional GUI-based travel planners.

    • What it does: Enhanced the list command to support dynamic sorting by name, start date, end date, and duration. It also introduced a "Temporal Dashboard" that summarizes trips into categories: Upcoming, Ongoing, Completed, and Planning.

Contributions to the User Guide

Listing all trips : list

Shows a list of all trips currently in the log and displays a Summary Dashboard in the result box. The list can be optionally sorted by a specific key.

The Summary Dashboard categorizes your trips based on the current date:

  • Upcoming: Trips starting after today.
  • Ongoing: Trips currently in progress.
  • Completed: Trips that have already ended.
  • Planning: Trips with no start date specified.

Format: list [sort/KEY] Example: list sort/len — Displays all trips ordered by duration (longest first).

Contributions to the Developer Guide

Trip Statistics and Multi-Key Sorting

The ListCommand bridges the Logic and Model layers to transform a simple list view into a temporal dashboard.

Sorting Mechanism & Tie-Breaking: The sorting is implemented using a Comparator Factory pattern within ListCommand. To ensure a stable user experience, a Multi-Level Tie-Breaker is used:

  1. Primary Key: User-selected key.
  2. Secondary Key: Case-insensitive alphabetical sort by trip name.

Temporal Dashboard Logic: The calculation is centralized in TripSummaryUtil#calculateSummary(). It determines status based on LocalDate.now():

  • Planning: startDate == null
  • Upcoming: today < startDate
  • Completed: today > endDate
  • Ongoing: startDate <= today <= endDate
