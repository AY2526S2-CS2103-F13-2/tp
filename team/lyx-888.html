Project Overview

TripLog is a CLI-centric desktop application designed for travelers who prefer managing their itineraries via a terminal interface. It allows users to track destinations and trip schedules, providing a streamlined, keyboard-first alternative to traditional GUI-based travel planners.

Deleting trip(s): delete

Confirmation required:
All delete operations first show a preview of the trips to be deleted.
Press Enter again to confirm the deletion, or edit the command to cancel.

Format:
delete INDEX
delete START-END
delete PREFIX/VALUE
delete sd/START_DATE ed/END_DATE

Contributions to the Developer Guide

Trip Deletion: Delete Command

The delete command supports multiple deletion modes, including index, range, field-based, and date-range deletion.

Parsing is handled by DeleteCommandParser, which determines the deletion mode based on input format and constructs the corresponding command.

A two-step confirmation mechanism is implemented:

  1. First execution shows a preview of affected trips
  2. Second execution confirms and performs deletion

Invalid combinations of delete modes are rejected during parsing to ensure correctness.
